# Documentação do ContactManager

Este diretório reúne os artefatos de documentação do projeto e serve como índice para os documentos principais.

## Documentos principais

* `requisitos.md` — Requisitos funcionais, não funcionais e restrições do sistema.
* `arquitetura.md` — Arquitetura do sistema e organização dos módulos em C.
* `casos-de-uso.md` — Casos de uso e diagramas de interação do usuário.
* `fluxos.md` — Fluxos de interação e operação do sistema.
* `modelos-de-dados.md` — Modelo de dados atual e formato de persistência.
* `product-backlog.md` — Backlog inicial do projeto.
* `testes.md` — Casos de teste planejados e status atual.
* `resultados.md` — Resultados concluídos até o momento.
* `diario/diario.md` — Diário de desenvolvimento com histórico por dia.

## Como usar

1. Leia `README.md` na raiz do projeto para entender o propósito geral.
2. Consulte `docs/README.md` para ver onde cada tipo de documentação está localizado.
3. Use `docs/requisitos.md` e `docs/arquitetura.md` como referências para implementação.
4. Atualize `docs/diario/diario.md` sempre que fizer mudanças significativas no projeto.

## Observações

A documentação foi reorganizada em junho de 2026 para refletir o estado atual da implementação em C e o ciclo de desenvolvimento do projeto.

O projeto está em evolução para uma arquitetura híbrida com PostgreSQL como backend planejado, Python para automações e uma interface gráfica futura.
